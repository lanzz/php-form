Class Form

Extends class Form_Container

This class represents the entire form. It is a Form_Container, because it contains the top-level Form_Element instances.

It provides several static methods to instantiate a Form from a given set of submitted values â€” either from actual GET/POST submitted values, or directly from an array of values.

Instantiation:

Instantiate a new form, populating it from GET query parameters:

$form = Form::from_get();

Instantiate a new form, populating it from POST parameters starting at $_POST['foo']['bar']:

$form = Form::from_post('foo.bar');

Rendering elements from the above form will likewise have names starting with foo[bar].

Instantiate a new form, populating it from both GET and POST parameters, POST parameters taking precedence in conflicts:

$form = Form::from_request();

Create a new form, populating it with preset values from an array:

$form = Form::from_array(array('foo' => 1, 'bar' => 2));

Create a new form, populating it from array, and namespacing it under foo:

$form = Form::from_array(array('bar' => 1, 'baz' => 2), 'foo');

When rendered, its elements will have names foo[bar] and foo[baz]

Using instances:

Set default values to populate the form elements before it has been submitted:

$form->set_defaults(array('foo' => 1, 'bar' => 2));

Do something only when the form has been actually submitted:

if ($form->is_submitted()) {
    // ...
}

Get a Form_Element instance for the foo element of the form:

$element = $form->foo;

Get a Form_Element instance for a deeper form element:

$element = $form->foo->bar[2]->baz;

This element would refer to the value of $_POST['foo']['bar'][2]['baz'].

Generate HTML with all the form's elements as type="hidden" inputs:

$html = $form->hidden();

Build a query string with all the form's elements' values:

$query = $form->query();
